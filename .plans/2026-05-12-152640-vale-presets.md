Date: 2026-05-12 15:26

## Goal

Record the Vale package review and decide which existing Vale rules should be integrated into `prosesmasher` presets.

This plan is only about Vale. It does not cover Rust heuristic development.

## Current Vale Scope

Vale is integrated as an optional external engine.

Current files:

- `vale/.vale.ini`
- `vale/harsh.ini`
- `vale/README.md`
- `apps/prosesmasher/crates/adapters/inbound/cli/runtime/src/vale.rs`
- `apps/prosesmasher/crates/adapters/inbound/cli/runtime/src/output.rs`
- `apps/prosesmasher/crates/domain/types/runtime/src/config.rs`
- `apps/prosesmasher/crates/adapters/outbound/fs/runtime/src/config_dto.rs`

Current config shape:

```json
{
  "externalEngines": {
    "vale": {
      "enabled": false,
      "binary": "vale",
      "configPath": "vale/.vale.ini",
      "minAlertLevel": "warning"
    }
  }
}
```

Current runner behavior:

- `prosesmasher` runs native checks first.
- If `externalEngines.vale.enabled` is true, it runs `vale --output JSON`.
- Vale findings are copied into `external_engines` in JSON output.
- Vale failures make the file fail.
- Vale rule behavior stays in Vale config files.
- Synced package files under `vale/styles/` should not be edited directly.

## What `alex` Does

`alex` catches insensitive or inconsiderate language. It is not a slop detector.

Rules found:

- `alex.ProfanityUnlikely`
- `alex.Condescending`
- `alex.Ablist`
- `alex.ProfanityMaybe`
- `alex.ProfanityLikely`
- `alex.Suicide`
- `alex.Gendered`
- plus unused or unhit rules in this corpus: `Press`, `OCD`, `LGBTQ`, `Race`

Fixture behavior:

- 1170 alerts across 133 files.
- Top counts:
  - `alex.ProfanityUnlikely`: 652
  - `alex.Condescending`: 246
  - `alex.Ablist`: 209
  - `alex.ProfanityMaybe`: 46
  - `alex.ProfanityLikely`: 14
  - `alex.Suicide`: 2
  - `alex.Gendered`: 1

Representative alerts:

- `easy`: "Using 'easy' may come across as condescending."
- `mental`: suggests alternatives when referring to a person.
- `stupid`: likely profanity.
- `trap`: possible profanity.
- `sick`: possible profanity in some cases.
- `failure`: possible profanity in some cases.
- `hang`: suggests app-specific alternatives because it may be insensitive.
- `masterpiece`: gendered-language warning.

Recommended `vale/inclusive.ini`:

```ini
StylesPath = styles
MinAlertLevel = warning

Packages = alex

[*.{md,mdx,txt}]
BasedOnStyles =
alex.ProfanityLikely = YES
alex.Suicide = YES
alex.Gendered = YES
alex.Race = YES
alex.LGBTQ = YES
alex.Press = YES
alex.OCD = YES
alex.Ablist = YES
alex.Condescending = NO
alex.ProfanityMaybe = NO
alex.ProfanityUnlikely = NO
```

Reason:

- `ProfanityLikely`, `Suicide`, `Gendered`, `Race`, `LGBTQ`, `Press`, `OCD`, and maybe `Ablist` are concrete enough to be useful.
- `Condescending` overflags ordinary words like `easy`.
- `ProfanityMaybe` and `ProfanityUnlikely` are too noisy for prose rewriting.

## Microsoft Rules Worth Considering

Full Microsoft package is too noisy.

Top fixture alerts:

- `Microsoft.Contractions`: 1667
- `Microsoft.Adverbs`: 551
- `Microsoft.Dashes`: 451
- `Microsoft.FirstPerson`: 315
- `Microsoft.We`: 272

Candidates:

- `Microsoft.Adverbs`
  - Examples: `broadly`, `quietly`, `seriously`, `deeply`
  - Message: remove if not important.
  - Good for harsh only.
- `Microsoft.Ellipses`
  - Example: `...`
  - Good for harsh if we want ASCII ellipses treated as sloppy punctuation.
- `Microsoft.Percentages`
  - Example: `twenty percent` -> numeral plus units.
  - Style-guide rule, not currently needed.
- `Microsoft.HeadingPunctuation`
  - Example: headings ending in `?`.
  - Probably not needed because native heading policy exists.
- `Microsoft.HeadingColons`
  - Heading colon capitalization.
  - Not needed for current prose target.
- `Microsoft.Hyphens`
  - Example: `early-morning` does not need a hyphen.
  - Too style-guide-specific for now.
- `Microsoft.Quotes`
  - US punctuation placement inside quotes.
  - Style preference only.
- `Microsoft.AMPM`
  - Example: `6 a.m.` -> `6 AM`.
  - Conflicts with normal editorial styles.

Avoid:

- `Microsoft.Contractions`
  - Forces `is not` -> `isn't`.
- `Microsoft.FirstPerson`
  - Overflags essays, posts, and personal writing.
- `Microsoft.We`
  - Overflags normal first-person plural.
- `Microsoft.Dashes`
  - Native em-dash checks already exist.
- `Microsoft.Avoid`
  - Example hit: `backbone`.
  - Too arbitrary without the Microsoft A-Z guide context.
- `Microsoft.Terms`
  - Example: prefers `phone` over `smartphone`.
  - Too domain-specific.
- `Microsoft.Units`
  - Example hit: `just pixels`.
  - Bad in general prose.

Recommended immediate Microsoft additions:

- none in `general`
- maybe `Microsoft.Adverbs = YES` in `vale/harsh.ini`
- maybe `Microsoft.Ellipses = YES` in `vale/harsh.ini`
