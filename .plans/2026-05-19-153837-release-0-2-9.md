# Goal

Release the widened slop rules as `slopless@0.2.9`.

# Approach

- Bump `package.json` and `package-lock.json` from `0.2.8` to `0.2.9`.
- Run local validation before pushing.
- Open a PR from `development` to `main`.
- Wait for GitHub CI to pass.
- Merge the PR.
- Publish `0.2.9` to npm from `main`.
- Create and push a `v0.2.9` git tag.
- Verify npm reports `0.2.9`.

# Files To Modify

- `package.json`
- `package-lock.json`
- `.plans/2026-05-19-153837-release-0-2-9.md`
- `.worklogs/<timestamp>-release-0-2-9.md`

# Verification

- `npm run validate`
- `gh pr checks`
- `npm view slopless version`
