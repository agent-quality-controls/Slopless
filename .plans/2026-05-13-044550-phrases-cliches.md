# Goal

Add the first external phrase enrichment as `phrases/cliches`.

The rule reports fixed multi-word cliches through textlint. It does not create a new `prose-lists` family.

# Source

- Source package: `no-cliches@0.3.0`
- Source license: MIT
- Dependency path: used by `write-good@1.0.8`
- Data source file: `no-cliches/cliches.js`

# Approach

- Record source attribution beside phrase data.
- Import the cliche list as JSON under `src/families/phrases/data/`.
- Add one textlint rule file: `src/families/phrases/cliches.ts`.
- Reuse token-boundary phrase matching.
- Add unquoted phrase matching in the shared matcher, so examples like `"at loose ends"` are not reported by this rule.
- Register the rule in the phrase registry and the `everything` preset.
- Add a phrase fixture trigger and regenerate the phrase baseline.
- Run the full fixture corpus before and after implementation and inspect new `cliches` findings.
- Commit only TypeScript phrase-rule work, fixture/baseline changes, plan, and worklog.
