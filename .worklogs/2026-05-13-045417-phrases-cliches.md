# Summary

Added `phrases/cliches` to the TypeScript textlint rules using the MIT `no-cliches@0.3.0` phrase list.

The rule uses existing token-boundary phrase matching, skips double-quoted examples, and is enabled in the `everything` preset.

# Decisions Made

- Put cliches under `phrases`, not a new `prose-lists` family, because the taxonomy owner is fixed multi-word expressions.
- Stored the imported cliche list as JSON data, not TypeScript source.
- Added source attribution beside the data.
- Removed `pain in the` from the imported source list because it caught literal medical prose.
- Ignored the imported cliche JSON in cspell instead of adding hundreds of source-list words to the dictionary.
