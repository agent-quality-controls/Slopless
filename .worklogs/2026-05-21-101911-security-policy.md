# Summary

Added a minimal GitHub security policy for Slopless. The policy supports only the latest released version and directs vulnerability reports to GitHub private vulnerability reporting.

# Decisions Made

- Used root-level `SECURITY.md` because GitHub recognizes that location directly.
- Kept support scope to the latest released version because the project does not maintain older release lines.
- Avoided publishing an email address or public issue workflow for vulnerability details.

# Key Files For Context

- `SECURITY.md`

# Next Steps

- If GitHub private vulnerability reporting is not enabled for the repository, enable it in GitHub repository security settings.
