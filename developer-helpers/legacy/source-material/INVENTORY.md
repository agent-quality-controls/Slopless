# Legacy Source Material Inventory

This folder contains reviewed records and raw provenance. Labeled data lives in `datasets/labeled`.

## reviewed/implemented

- Contents: incorporation records and implementation batch notes.
- State: implemented in Slopless, usually with fixtures and Fixture3 golden output.
- Use: explain why source material became a rule, lexicon, pattern, or fixture.

## reviewed/skipped

- Contents: candidate notes reviewed and not queued.
- State: not implemented.
- Use: preserve rejection reasons, false-positive risks, and source context.

## reviewed/deferred

- Contents: candidates deliberately delayed.
- State: not implemented yet.
- Current deferred group: Markdown-layout artifacts.

## raw-provenance

- Contents: raw HTML pages, upstream readmes, API captures, extraction reports, PDFs, and other source evidence.
- State: provenance only.
- Use: verify where datasets and rule ideas came from.
